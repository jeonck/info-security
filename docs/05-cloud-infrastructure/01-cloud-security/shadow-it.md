# 섀도우 IT (Shadow IT)

## I. 기업 보안의 보이지 않는 위협, 섀도우 IT의 개요

**정의**: 기업의 정보보호 정책에 반하거나 공식적인 자산 관리 프로세스를 거치지 않고 업무에 활용되는 정보자산 및 서비스

**발생 원인**: IT 부서의 느린 피드백(업무 병목), 클라우드 기반 SaaS의 도입 용이성, 사용자 편의성 우선주의 및 원격 근무 확산

---

## II. 섀도우 IT의 위험 유형 및 관리 기술

### 가. 섀도우 IT의 주요 위험 유형

- **데이터 유출**: 개인용 클라우드 스토리지(Dropbox, Google Drive 등)를 통한 기업 기밀 유출
- **컴플라이언스 위반**: 개인정보가 포함된 데이터를 보안 인증을 받지 않은 외부 서비스에서 처리
- **보안 취약점**: 보안 패치가 이루어지지 않은 구형 장비나 보안 기능이 미흡한 무료 협업 도구 사용

### 나. 기술적 탐지 및 관리 방안

| 관리 기술 | 상세 설명 | 비고 |
|----------|---------|------|
| **CASB** | 클라우드 서비스 이용 현황을 가시화하고 보안 정책 적용 | SaaS 관리에 최적화 |
| **DLP** | 민감 데이터가 비인가 경로로 유출되는 것을 실시간 차단 | 데이터 중심 통제 |
| **NGFW** / **SWG** | 네트워크 게이트웨이에서 비인가 애플리케이션 트래픽 탐지 | 프로토콜 기반 식별 |
| **EASM** | 외부로 노출된 자산(Attack Surface)을 자동으로 식별 | 외부 공격 표면 관리 |

## III. 섀도우 IT 대응을 위한 거버넌스 전략 비교

| 구분 | 통제 중심 (Control-focused) | 수용 중심 (Enabling-focused) |
|------|---------------------------|----------------------------|
| **전략 방향** | 비인가 서비스의 원천적 차단 및 금지 | 보안 요건 충족 시 공식 자산으로 수용 |
| **장점** | 보안 리스크의 확실한 제거 | 업무 생산성 향상 및 개발 민첩성 확보 |
| **단점** | 사용자 불편 가중, 우회 경로 발생 위험 | 관리 포인트 증가 및 보안 비용 상승 |
| **적용 사례** | 망 분리 환경, 금융/국방 등 폐쇄망 | 스타트업, 일반 기업의 유연한 IT 환경 |

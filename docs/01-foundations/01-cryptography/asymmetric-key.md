# 신뢰 기반의 이중키 체계, 비대칭키 암호

## I. 비대칭키 암호의 정의 및 특징

**정의**: 수학적 난제(소인수분해, 이산대수 등)를 기반으로 생성된 **공개키**(Public Key)와 **개인키**(Private Key) 쌍을 이용하는 암호 방식

**특징**: 키 분배가 용이하고 기밀성뿐만 아니라 부인 방지(Non-repudiation) 기능을 제공하나, 대칭키 대비 연산 속도가 느림

---

## II. 비대칭키 암호의 작동 메커니즘

### 가. 기밀성과 인증(전자서명) 프로세스

```mermaid
graph TD
    subgraph "기밀성 (Confidentiality)"
        A1["송신자"] -->|"수신자의 공개키로 암호화"| B1["암호문"]
        B1 -->|"수신자의 개인키로 복호화"| C1["수신자 (기밀성 확보)"]
    end

    subgraph "인증 및 부인방지 (Authentication)"
        A2["송신자"] -->|"송신자의 개인키로 암호화"| B2["전자서명"]
        B2 -->|"송신자의 공개키로 복호화"| C2["검증 (신원 확인)"]
    end
```

**상세 메커니즘**:
- **기밀성**(Confidentiality): 수신자의 공개키로 암호화 → 수신자의 개인키로만 복호화 가능
- **인증 및 부인방지**(Authentication): 송신자의 개인키로 암호화(서명) → 송신자의 공개키로 누구나 복호화(검증)

### 나. 주요 알고리즘 및 수학적 난제

| 알고리즘 | 기반 수학적 난제 | 특징 및 용도 |
|----------|----------------|-------------|
| **RSA** | 소인수분해 (Integer Factorization) | 가장 널리 사용됨, 키 길이가 길어지는 추세 (2048bit 이상) |
| **ECC** | 타원곡선 이산대수 (ECDLP) | RSA보다 짧은 키로 동일 보안 강도 제공 (모바일 / IoT 최적) |
| **Diffie-Hellman** | 이산대수 (Discrete Logarithm) | 키 교환 전용 알고리즘, SSL / TLS 초기 단계 사용 |
| **ElGamal** | 이산대수 | 암호문이 평문의 2배 크기로 증가하는 단점 존재 |

## III. 대칭키 암호와의 비교 및 하이브리드 활용

| 비교 항목 | 대칭키 암호 | 비대칭키 암호 |
|----------|------------|--------------|
| **키 개수** | 1개 (공유 비밀키) | 2개 (공개키, 개인키) |
| **키 분배** | 어려움 (사전 공유 필요) | 매우 용이 (공개키 배포) |
| **연산 속도** | 빠름 (대용량 처리) | 느림 (약 100 ~ 1,000배 차이) |
| **핵심 용도** | 데이터 본체 암호화 | 키 교환, 전자서명, 인증 |

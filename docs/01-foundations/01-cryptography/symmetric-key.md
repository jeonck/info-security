# 고속 데이터 보호의 핵심, 대칭키 암호

## I. 대칭키 암호의 정의 및 특징

**정의**: 송신자와 수신자가 사전에 공유된 **단일 비밀키**(Secret Key)를 사용하여 평문을 암호문으로 바꾸고, 다시 평문으로 되돌리는 암호 체계

**특징**: 비대칭키(공개키) 암호에 비해 연산 속도가 매우 빠르며, 알고리즘 구조가 상대적으로 단순하여 대용량 데이터 암호화에 효율적임

---

## II. 대칭키 암호의 메커니즘 및 분류

### 가. 대칭키 암호 방식의 구조

- **암호화**: `C = E(K, P)` (P: 평문, K: 공유키, C: 암호문)
- **복호화**: `P = D(K, C)` (암호화와 복호화에 동일한 K 사용)

### 나. 데이터 처리 단위에 따른 분류

| 구분 | 스트림 암호 (Stream Cipher) | 블록 암호 (Block Cipher) |
|------|---------------------------|------------------------|
| **처리 단위** | 1비트(Bit) 또는 1바이트(Byte) 단위 | 고정된 크기의 블록 (64 / 128 / 256 Bit) |
| **핵심 원리** | 유사난수 스트림과 XOR 연산 | **치환**(S)과 **순열**(P)의 반복 |
| **연산 속도** | 매우 빠름 (실시간성) | 스트림 대비 상대적으로 느림 |
| **주요 사례** | RC4, A5 / 1, Salsa20 | DES, AES, SEED, ARIA |
| **장단점** | 하드웨어 구현 용이, 낮은 확산성 | 높은 보안성, 패딩(Padding) 필요 |

---

## III. 대칭키 암호의 한계점 및 해결 방안

| 한계점 (Issues) | 상세 내용 | 해결 방안 |
|:---:|----------|----------|
| **키 분배 문제** | 수신자에게 키를 안전하게 전달하기 어려움 | Diffie-Hellman, 하이브리드 암호(봉투) |
| **키 관리 부담** | $n$명의 사용자 사이에서 $n(n-1)/2$개의 키 필요 | KDC(키 분배 센터), 공인인증 체계 연계 |
| **무결성 부재** | 키 노출 시 기밀성뿐만 아니라 위변조 위협 | HMAC, GCM 모드 등 인증 암호 활용 |

# 자원 보호의 첫걸음, 접근 제어 (Access Control)

## I. 접근 제어의 정의 및 3대 요소

**정의**: 주체(Subject)가 객체(Object)에 접근할 때, 적절한 권한이 있는지 확인하고 이를 허용하거나 거부하는 정책 및 기술적 수단

**3대 요소**:
- **식별**(Identification): 주체가 누구인지 주장하는 과정 (ID 입력)
- **인증**(Authentication): 주체의 주장이 맞는지 증명 (Password, 생체인증)
- **인가**(Authorization): 검증된 주체에게 특정 자원 사용 권한을 부여

---

## II. 접근 제어의 3가지 정책 모델 및 비교

### 가. 주요 접근 제어 모델 구조

| 구분 | 임의적 접근제어 (DAC) | 강제적 접근제어 (MAC) | 역할기반 접근제어 (RBAC) |
|------|--------------------|--------------------|----------------------|
| **제어 주체** | 자원 소유자 (Owner) | 관리자 / 시스템 (Admin) | 중앙 관리자 |
| **판단 근거** | 주체의 신분 (Identity) | 보안 등급 (Label) | 사용자의 역할 (Role) |
| **특징** | 구현 용이, 유연함 | 가장 강력한 보안 | 관리 효율성 높음 |
| **장점** | 사용자의 편의성 극대화 | 데이터 무결성 / 기밀성 보장 | 사용자 이동 / 복직 관리 용이 |
| **단점** | 트로이 목마 등 취약 | 구현 복잡, 사용자 불편 | 역할 설계의 초기 비용 발생 |
| **사례** | Windows 파일 권한, ACL | 군사 시스템, 국방 보안 | 기업용 ERP, 일반 기업 |

### 나. 접근 제어의 기술적 메커니즘

- **접근 제어 행렬**(Access Control Matrix): 주체와 객체를 행과 열로 구성하여 권한을 기록한 테이블
- **CL**(Capability List): 주체 관점에서 접근 가능한 객체 목록을 관리 (Subject 중심)
- **ACL**(Access Control List): 객체 관점에서 접근 가능한 주체 목록을 관리 (Object 중심)
